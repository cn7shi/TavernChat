# 2026-07-22

## IM 可靠性传输代码评审
- 评审了 `src/main/kotlin/com/tavernchat/core/{channel,model,reliability}` 的可靠传输引擎。
- 结论：ACK/重传/去重/乱序重组 happy-path 已实现（测试全绿），但离线消息、断线恢复、持久化三块基本空白。
- 关键缺陷：
  1. `ReliableSender.sendReliable` 先 `channel.send` 后注册 `pendingAcks`，同步通道下首包 ACK 被竞态丢弃，必然多一次重传。
  2. `seqId` 由调用方生成、未托管；reorderer 单一全局序列流，未按 (sender,conversation) 隔离，多发送方会排序错乱。
  3. 去重 LRU 窗口仅 50，窗口外重复会重新交付；`Message.id` 默认值空串，依赖调用方生成唯一 id。
- 产出评审报告：`docs/IM可靠性传输评审.md`。
